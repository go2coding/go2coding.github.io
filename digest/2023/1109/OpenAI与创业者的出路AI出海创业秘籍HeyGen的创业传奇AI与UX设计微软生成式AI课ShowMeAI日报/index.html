yusen 将 OpenAI 与PC时代的 Intel 和 Microsoft 进行类比，提出了三种可能的赢家策略：

  1. 类似于 Character，在一个细分市场里大幅超过 OpenAI，然后围绕独特的模型能力打造独特的产品和生态

  1. 莫慌！OpenAI 当前在中间层和应用层的试探仍然不够成熟，Agent 也还不可靠 ，中间层公司仍然有无可替代的重要作用
  1. OpenAI 这次并没有发布颠覆性的新产品，只是取代了一些价值有限的套壳类产品和初级 Agent 工具 ；如果有创业公司因此被影响，说明其产品价值的护城河本来就不牢固

👀 马斯克 xAI 发布 PromptIDE

https://x.ai/prompt-ide

PromptIDE 是一个由 xAI 开发的集成开发环境 (IDE)，专为提示工程和可解释性研究而设计。 它提供了一个 Python 代码编辑器和 SDK，帮助用户实现复杂的提示技术并分析模型输出，还支持用户输入、文件处理和并发处理，主要特点包括：

1.透明访问 Grok-1 模型 ：PromptIDE 允许用户在 xAI 社区中透明地访问 Grok-1 模型，探索大型语言模型 (LLMs) 的能力

2.代码编辑器与 SDK ：PromptIDE 的核心是一个 Python 代码编辑器，与一个 SDK 相结合，允许用户实现复杂的提示技术

3.并发处理 ：SDK 使用 Python 协程，允许同时处理多个注释的 Python 函数，从而显著提高完成时间，特别是在处理 CSV 文件时

4.用户输入 ：Prompts 可以通过 user_input() 函数实现交互式输入，该函数在 UI 中显示一个文本框，等待用户输入

5.文件处理 ：开发者可以上传小文件 (最多 5 MiB 每文件，总共最多 50 MiB)，并使用上传的文件在提示中进行处理

6.分析功能 ：在执行提示时，用户可以看到详细的逐令牌分析，以帮助他们更好地理解模型的输出

👀 AI与UX设计：提高生产力与创造力的秘诀

https://www.nngroup.com/articles/ai-ux-getting-started

UX (User Experience / 用户体验) 是指用户在使用产品、服务或系统过程中的感受、情感和认

NNGroup (Nielsen Norman Group) 是一家全球领先的用户体验 (UX) 研究和咨询公司，由 Jakob Nielsen 和 Don Norman 于1998年创立

这是 NNGroup 最新发布的「AI+设计」系列文章的入门篇，为既羡慕 AI 又担心其代替自己工作的设计师们，普及了在 UX 设计中使用 AI 的基本思路。设计从业者或者有意参与AI创业的伙伴们，可以看看设计行业顶尖专家的见解：

1.为什么在UX工作中使用AI ：可以更高效地完成任务，提高工作质量，并利用AI工具来增强自己的技能

2.AI对经验丰富的UX专业人士是安全的 ：所有UX专业人士都应将AI加入工作流，但人类也需参与识别幻觉和处理偏见

3.AI工具推荐 ：ChatGPT是一个强大的文本生成AI工具，而Midjourney则擅长生成高质量的图像

4.使用AI进行UX设计的具体建议 ：包括提供足够的上下文、要求多个选项、迭代输出和建立提示库

5.列举AI可以提供帮助的UX任务 ：AI可以用于生成设计想法、创建插图、编写用户访谈问题等

6.AI不能替代设计师、研究人员或用户 ：AI生成的设计建议可能是错误的，而AI不能替代真实的用户研究

7.UX专业人士不需要深入了解AI的工作原理 ：但了解AI的基本概念和底层技术是有益的，有助于更好地沟通与技术同事，并找到克服AI局限性的方法

8.保持信息获取和知识体系更新 ：推荐订阅一些关于AI的时事通讯，以便了解AI的最新发展

9.立即开始，从小处着手 ：建议从现在开始使用AI，从小型任务开始并逐渐应用于更复杂的UX任务

🉑 微软发布「面向初学者的生成式AI课」，12节课带你构建AI应用

https://github.com/microsoft/generative-ai-for-beginners

微软推出了一门专为初学者打造的课程「Generative AI for Beginners」 ，该课程总共 12 课时，每节课包含一个简短的主题视频介绍。课中讲解的项目还会有 Juypter Notebook 代码示例，并且用户还能链接到额外的资源，进行更深入的学习。

1.生成式AI和 LLMs 简介 ：了解生成式AI是什么以及大型语言模型 (LLMs) 的工作原理

2.探索和比较不同的 LLMs ：测试、迭代和比较不同的大语言模型

3.负责任地使用生成式AI ：理解基础模型的限制和AI背后的风险

4.理解提示工程基础 ：讲解提示工程最佳实践的应用，理解提示结构和用法

5.创建高级提示 ：扩展对提示工程的认知，使用不同技术来改进提示的结果

6.构建文本生成应用 ：使用 Azure OpenAI 构建文本生成应用，了解如何有效地使用令牌和温度来改变模型的输出

7.构建聊天应用 ：高效构建和集成聊天应用的技术，确定有效监控和维护AI驱动聊天应用质量的关键指标和考虑因素

8.构建搜索应用向量数据库 ：语义 VS 关键字搜索，创建一个使用 Embeddings 搜索数据的应用程序

9.构建图像生成应用 ：讲解图像生成原理，以及如何构建图像生成应用

10.构建低代码AI应用 ：使用低代码为教育初创公司构建学生作业跟踪应用程序

11.集成外部应用程序与函数调用 ：功能调用及其在应用中的应用案例

12.设计AI应用的用户体验 ：开发者在开发生成式 AI 应用程序时，帮助其使用 UX 设计原则
